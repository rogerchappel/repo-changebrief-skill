# repo-changebrief-skill PRD

## Objective
Help agents turn repository work into accurate, evidence-backed launch material without inventing claims.

## Users
- Agent builders preparing release candidates.
- Maintainers who need quick demos, posts, and risk notes from a change summary.

## MVP
- Parse Markdown or JSON change summaries.
- Extract files, verification, artifacts, risks, and audience hints.
- Classify the change as feature, fix, docs, test, chore, or mixed.
- Render release notes, demo outline, post draft, and follow-up checklist.

## Non-goals
- Publishing posts or releases.
- Calling external APIs.
- Replacing human review of claims.
